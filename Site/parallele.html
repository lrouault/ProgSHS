Parallèle

Le code de calcul que nous avons réalisé est destiné à être utilisé sur des modèles 3D avec une technique de raffinement de maillage adaptatif (AMR). Pour ces raisons, il est nécessaire de paralléliser et optimiser le code de manière à accélerer son exécution.

Découpe du maillage

2D OOOOOOO

Pour paralléliser l'éxecution du code, nous utilisons la bibliothèque MPI. Il faut découper le maillage de sorte que chaque processus qui exécute le code ait la même charge de travail. On essaye donc de faire en sorte que chaque processus ait à traiter le même nombre de points de maillage.

Pour cela, il y a plusieurs solutions. On peut choisir de diviser le nombre total de points de maillage par le nombre de processus et répartir ces points entre les différents processus. Mais dans notre cas, puisque le code est destiné à fonctionner avec une technique d'AMR, nous découpons plutôt le maillage en bandes afin que chaque processus ait la même charge de travail étant donné qu'avec la technique d'AMR, certaines zones du maillage contiennent plus de points de maillages que d'autres. Voici un exemple ci-dessous qui montre comment est découpé le maillage en 2D où chaque couleur représente un processus différent.

3D

De la même façon qu'en 2D, on découpe le maillage 3D en bandes.
